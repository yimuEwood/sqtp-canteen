常见问题：

1. 错误：new row violates row-level security policy

原因：profiles表中该用户的role不是'admin'

解决：去Supabase Dashboard -> Table Editor -> profiles表，检查管理员账号的role字段

2. 错误：duplicate key

原因：插入时指定了ID但ID已存在

解决：代码中不应手动指定ID，让数据库自动生成

Supabase Dashboard 检查步骤：

  1. 打开 Supabase Dashboard
  2. 选择项目 imewztdgharjzcrwibsa
  3. 左侧菜单选择 Table Editor
  4. 点击 profiles 表
  5. 找到您的管理员账号，确认 role 列显示为 admin
  6. 如果没有记录或role不对，需要修复

快速修复 SQL（需要执行）：

如果profiles表中管理员记录缺失或role不对，在Supabase SQL Editor中执行：

-- 先查询管理员邮箱对应的user_id
SELECT id, email FROM auth.users WHERE email = '3240103558@zju.edu.cn';

-- 然后更新或插入profiles记录
INSERT INTO public.profiles (id, email, role)
VALUES ('上面查询到的user_id', '3240103558@zju.edu.cn', 'admin')
ON CONFLICT (id) DO UPDATE SET role = 'admin';
